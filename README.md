# DrPlaga.sh

Cybersecurity lab focused on Threat Intelligence, defensive research, CTF practice, automation and hardware hacking.

I use this space to document my learning path, publish lab-based research, build small security tools and share defensive analysis with evidence.

## Focus areas

- Threat Intelligence / CTI
- Defensive security research
- CTF writeups and lab machines
- CVE analysis in controlled environments
- Bash and Python automation
- Linux, networking and infrastructure
- Hardware hacking, SDR, UART, JTAG/SWD
- Cryptography labs and security notes

## Main repositories

- `ctf-writeups`  
  Writeups from CTF platforms, vulnerable machines and local labs.

- `cve-lab-pocs`  
  CVE analysis and proof-of-concept code tested only in controlled environments.

- `security-tools`  
  Personal scripts and utilities for recon, parsing, automation and defensive workflows.

- `crypto-labs`  
  Cryptography exercises, examples and notes.

- `hardware-hacking-lab`  
  Notes, drivers, experiments and procedures related to hardware security.

- `payload-analysis`  
  Defensive analysis of suspicious payloads, decoding notes, IOCs and detection logic.

## Principles

- Evidence before noise.
- Defensive use only.
- No credentials, tokens, dumps or sensitive data.
- No attacks against third-party systems.
- Everything published here is for authorized labs, education and research.

## Current interests

- Honeypots as a source of Threat Intelligence
- Automation for defensive enrichment
- CVE tracking and practical mitigation
- SOC workflows with Wazuh, OpenCTI and related tools
- Infrastructure segmentation and resilient lab design
