# Security Policy

## Scope

This GitHub profile and its repositories are focused on cybersecurity learning, defensive research, CTF practice, lab-based CVE analysis, payload analysis and security tooling.

The content published here is intended for:

* Authorized labs
* CTF environments
* Defensive research
* Educational use
* Controlled testing environments

## Reporting security issues

If you find a security issue related to one of my repositories, please report it responsibly.

Do not open a public issue if the report contains:

* Secrets
* Tokens
* Credentials
* Sensitive data
* Exploit details that could enable abuse
* Private infrastructure information

Use a private contact method instead.

## What to include

When reporting an issue, include the following information:

* Repository
* Affected file
* Issue summary
* Impact
* Steps to reproduce
* Suggested fix
* Whether sensitive data is involved

Example report format:

```text
Repository:
Affected file:
Issue summary:
Impact:
Steps to reproduce:
Suggested fix:
Sensitive data involved: yes/no
```

## Out of scope

The following are out of scope:

* Attacks against third-party systems
* Reports based on unauthorized testing
* Social engineering
* Credential theft
* Denial of service
* Spam or automated noise
* Reports involving leaked data or dumps
* Requests to run tools against real targets
* Requests to publish live malware or weaponized payloads
* Reports involving systems outside my repositories or lab material

## Safe handling

Please do not send:

* Live malware
* Stolen credentials
* Dumps
* Tokens
* Private keys
* Session cookies
* Personal data
* Sensitive screenshots

If a sample or technical artifact is needed, provide safe alternatives:

* Hashes
* Sanitized snippets
* Redacted logs
* Safe reproduction notes
* Detection context
* Screenshots with sensitive data removed

## Repository content expectations

Public repositories under this profile should follow these principles:

* Defensive use only
* Authorized labs only
* No third-party targeting
* No hardcoded secrets
* No credential theft
* No persistence or evasion tooling
* No live malware binaries
* No leaked data
* No unsafe publication of payloads

## Responsible use

The material published in these repositories may include security research, CTF notes, lab-only PoCs, payload analysis, scripts and defensive tooling.

This material must not be used for:

* Unauthorized access
* Disruption of services
* Credential theft
* Persistence
* Evasion
* Malware deployment
* Attacks against systems without explicit permission

## Disclaimer

All content in these repositories is provided for educational, defensive and authorized lab use only.

Do not use this material against systems you do not own or do not have explicit permission to test.
